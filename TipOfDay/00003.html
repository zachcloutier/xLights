Troubleshooting why the visualiser won't work

It is not always clear why sometimes the visualiser button is enabled and working on the controller tab and other times it is disabled so lets run through what needs to be true for the visualiser button to work.

  • Output to lights must be turned off. The visualiser cannot be used if you are outputting to lights.
  • The controller must have the "Vendor", "Model" and when it appears the "Variant" properties set to a valid value. Without these set the visualiser does not know about your controller and thus is unable to present the number of ports or enforce the limits of the controller.
  • The controller must not be a NULL controller. To be honest there are not many reasons these days why a NULL controller is even required today other than maybe an xSchedule virtual matrix.

While not a strict requirment we would also recommend that "Auto Layout Models" and "Auto Size" also be selected. These allow xLights to juggle the model start channels to be optimised to work with your controller.
